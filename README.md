# 🗺️ 泳道图架构图一句话生成

> **美观、专业、一句话出图**  
> 横向泳道 · 竖向泳道 · 分层架构，Draw.io XML 即下即用

---

## ✨ 三大核心能力

### 🎯 横向泳道图
```
多角色业务流转 | 自上而下时序推进 | 默认方向
```
适用于：用户注册流程、订单处理、审批流转等典型业务流程

**效果示例**：4角色横向泳道，绿色起点→节点→红色终点，专业配色
```
┌─────────┬─────────┬─────────┬─────────┐
│  用户   │ 订单系统 │ 库存系统 │ 支付系统 │
├─────────┼─────────┼─────────┼─────────┤
│  (开始) │         │         │         │
│    ↓    │         │         │         │
│  下订单 │         │         │         │
│    ↓    │  扣库存 │         │         │
│    ↓    │    ↓    │  成功   │         │
│    ↓    │  发起支付←←←←←←←←←←←│
│    ↓    │    ↓    │         │  验证   │
│  完成   │  通知发货│         │         │
└─────────┴─────────┴─────────┴─────────┘
```

### 🎯 竖向泳道图
```
多角色并行对比 | 从左到右分区展示 | 自动推荐
```
适用于：政务多部门会签、企业跨部门协作、角色分工明确的场景

**效果示例**：6部门竖向并行对比，标签在顶部
```
┌───────┬───────┬───────┬───────┬───────┬───────┐
│申请人 │ 窗口  │  科室  │ 法制  │ 领导  │ 第三方│
├───────┼───────┼───────┼───────┼───────┼───────┤
│提交材料│ 受理  │ 初审  │ 审核  │ 审批  │ 评估  │
│  ↓    │  ↓    │       │       │       │  ↓    │
│       │ 补正  │       │       │       │       │
│       │  ↓    │       │       │       │       │
└───────┴───────┴───────┴───────┴───────┴───────┘
```


**效果示例**：电商微服务分层架构

## 🎨 专业配色方案

| 配色主题 | 说明 |
|---------|------|
| **政务蓝灰系** | 适合政府、事业单位场景，稳重专业 |
| **科技蓝白系** | 适合互联网、科技公司，现代简洁 |
| **商务深灰系** | 适合金融、咨询机构，高端大气 |
| **清新绿白系** | 适合医疗、教育机构，温和友好 |

**节点配色规范**：
- 开始节点：`#82b366` 绿色实心圆
- 结束节点：`#d9574a` 红色圆环
- 活动节点：`#dae8fc` 蓝底圆角矩形
- 判断节点：`#fff2cc` 黄底菱形
- 子流程：`#f8cecc` 红边双线矩形

## 🚀 一句话出图体验

### 输入示例
```
"画一个电商订单处理流程图，用户、订单、库存、支付4个系统"
```
```
"做个政务服务事项办理泳道图，包含申请人和4个审批部门"
```
```
"生成一个微服务架构图，包含用户、订单、支付三个核心服务"
```

### 输出
- **Draw.io XML** `.drawio` 文件，可直接在 [diagrams.net](https://app.diagrams.net) 打开编辑
- **ProcessOn**（可选配置 API Key），生成精美云端图表

---

## 📦 输出格式

### Draw.io XML（默认）
```xml
<?xml version="1.0" encoding="UTF-8"?>
<mxGraphModel>
  <root>
    <mxCell id="0"/>
    <mxCell id="1" parent="0"/>
    <!-- 泳道和节点定义 -->
  </root>
</mxGraphModel>
```
💡 保存为 `.drawio` 文件，拖入 [diagrams.net](https://app.diagrams.net) 即可编辑

- **专业格式输出**：Draw.io XML 标准格式，兼容所有流程图工具
